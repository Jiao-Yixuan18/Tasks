#### 2.你认为使用 Git Bash 提交麻烦吗？找一找，有没有更方便、更有效率的提交方式？

我认为Git Bash 提交比较麻烦，需要写很多命令。通过AI CSDN google浏览器等工具，我了解到还有以下更高效的提交方式：
* GitHub Desktop
1. 安装 GitHub Desktop
2. 打开仓库
3. 左侧看到修改的文件
4. 填写提交信息
5. 点击按钮提交并推送
*  VS Code
1. 在 VS Code 中打开项目
2. 左侧点击源代码管理图标
3. 看到所有修改的文件
4. 点击 + 号暂存更改
5. 输入提交信息，点击提交
6. 点击同步更改推送
* sourcetree
1.在Sorcetree主页面点击仓库
2.查看文件状态
3.暂存文件：在“File Status”标签页看到所有修改的文件，勾选单个文件（想要提交的文件）前的复选框，或点击“Stage All”暂存所有修改
4.填写提交信息：在下方“Commit”区域

第一行：添加hello.md个人介绍文件

下方描述：
创建hello.md文件
包含个人姓名介绍
用于Git学习任务
5.提交到本地仓库：点击“Commit”按钮
6.推送到远程仓库：点击“Push”,选择要推送的分支，点击“OK”完成
